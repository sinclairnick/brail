# Brail Devtools

<Callout type="warning">
The below applies only to next.js applications. **Devtools support will be expanded in the future.**
</Callout>

The Brail Devtools make working with your templates easier, but are not strictly necessary.

The Devtools consists of a frontend and backend component. Both require access to your templates, so we first initialise the devtools, which returns both pieces.

```tsx
/* ðŸ“„ lib/devtools.ts */
import { initNextDevtools } from "brail/devtools";

export const {
  DevtoolsLayout, // For frontend
  devtoolsHandler, // For backend
} = initNextDevtools({ 
	templates, /** ðŸ‘ˆ Register your templates with the devtools */
});

/* ðŸ“„ pages/api/devtools/[trpc].ts */
export default devtoolsHandler(); /** ðŸ‘ˆ Export API handler, to enable e.g. sending test emails */

/* ðŸ“„ pages/_app.tsx */
export default (props: AppProps) => {
  const { Component, pageProps } = props;
  return (
    <DevtoolsLayout>
      {/* ðŸ‘† Wrap devtools around app. This will not affect generated template HTML */}
      <Component {...pageProps} />
    </DevtoolsLayout>
  );
};
```

To ensure routing between templates works, we need to register the brail next.js config.

```tsx
const withBrail = require("brail/next")(/docs/* options */);

module.exports = withBrail({ /** ðŸ‘ˆ Add Brail plugin */
	/* Your next.js config */
});
```
