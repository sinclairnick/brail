The Brail 'devtools' make working with your templates easier, but are not strictly necessary.

The devtools consists of a frontend and backend component. Both require access to your templates, so we first initialise the devtools, which returns both pieces.

```tsx
//---| lib/devtools.ts |---
import { initNextDevtools } from "brail/devtools";

const templates = {
  /* Your templates */
};

export const {
  DevtoolsLayout, // For frontend
  devtoolsHandler, // For backend
} = initNextDevtools({
  templates,
});

//---| pages/api/devtools/[trpc].ts |---
export default devtoolsHandler();

//---| pages/_app.tsx |---
export default (props: AppProps) => {
  const { Component, pageProps } = props;
  return (
    <DevtoolsLayout>
      {/* Wrap devtools around app. This will not affect generated template HTML */}
      <Component {...pageProps} />
    </DevtoolsLayout>
  );
};
```

To ensure routing between templates works, we need to register the brail next.js config.

```tsx
const withBrail = require("brail/next")(/docs/* options */);

module.exports = withBrail({
	/* Your next.js config */
});
```

For more information, see the [**next.js integration docs**](/docs/recipes/nextjs)
